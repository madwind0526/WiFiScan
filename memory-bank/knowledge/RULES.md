# Rules

## Project Independence

- WifiScan은 기존 프로젝트와 분리된 새 프로젝트다.
- 다른 프로젝트에서는 폴더 구조와 검증된 작업 절차만 참고한다.
- 기능, 데이터 모델, 패키지, 비즈니스 규칙은 WifiScan 요구사항에 근거해 새로 결정한다.

## Source and UI

- 코드 주석은 영어만 사용한다.
- 사용자에게 보이는 UI 텍스트는 한국어로 작성한다.

## Discovery Completeness

- 단일 단말 스캔으로 모든 장비 탐지를 보장하지 않는다.
- 공유기, 서브넷, 서비스 탐색 결과를 결합하고 관측 소스, 신뢰도, 누락 가능성을 표시한다.
- 장비가 탐지되지 않았다는 사실을 안전하다는 의미로 표현하지 않는다.

## Safe Remediation

- 비밀번호 대입, 취약점 악용, 서비스 중단, 무단 패치를 구현하지 않는다.
- 자동 대응은 공식 관리 API, 사용자 인증, 변경 미리보기, 명시적 확인, 실패 복구 경로가 모두 있을 때만 허용한다.
- 제품과 버전이 확인되지 않은 장비에는 특정 취약점 판정을 확정하지 않는다.

## Android Permissions

- Android 17/API 37 이상에서 직접 LAN 통신을 사용하면 `ACCESS_LOCAL_NETWORK` 런타임 권한 또는 시스템 중개 탐색 경로를 사용한다.
- Wi-Fi API 권한은 Android 버전과 실제 호출 API별로 분리하고 거부/철회 상태를 정상 흐름으로 처리한다.

## Active Discovery Limits

- 큰 서브넷은 무제한 탐색하지 않는다.
- Windows Wave 2 능동 탐색은 현재 주소가 속한 최대 `/24`와 254개 호스트로 제한하고 UI에 범위 제한을 표시한다.
- 식별자(IP, MAC 등)는 테스트와 로그에 평문으로 출력하지 않는다.

## Inventory Storage

- 장비 스냅샷은 기본적으로 로컬 애플리케이션 지원 디렉터리에만 저장한다.
- 첫 검색은 기준선이며 신규 장비 경고를 발생시키지 않는다.
- 로컬 스냅샷은 무제한으로 쌓지 않고 최근 30개만 유지한다.

## Remediation Safety

- 공식 공유기 커넥터가 없으면 자동 차단·격리·설정 변경을 실행하지 않고 수동 안내만 제공한다.
- 커넥터 실행에는 사용자 명시 승인, 변경 미리보기, 확인 문구, 복구 가능성이 필요하다.

## Android Verification

- Android 권한 요청 전 한국어 근거 화면을 제공한다.
- 권한 거부/철회 상태에서도 앱이 종료되지 않고 안내 상태를 유지한다.
- 실기기 미연결 시 Android APK 빌드와 플랫폼 채널 코드 검증만으로 완료를 주장하지 않는다.

## Profile Credential Storage

- 프로필 메타데이터 파일에는 Wi-Fi 암호나 앱 고정 키 암호문을 저장하지 않는다.
- Wi-Fi 암호는 운영체제 보안 저장소에 프로필별로 보관하며, 프로필 삭제 시 대응하는 보안 저장소 항목도 삭제한다.
- 기존 `passwordEnc` 값은 보안 저장소 이전이 성공한 뒤에만 메타데이터 파일에서 제거한다.
- 프로필 내보내기 암호는 8자 이상으로 입력·확인하고 앱에 저장하지 않는다.
- 내보내기 파일은 무작위 salt/nonce, PBKDF2-HMAC-SHA256, AES-256-GCM을 사용하는 버전형 암호화 봉투로 만든다.
- WifiScan에서 삭제한 Windows 자동 탐색 프로필은 재시작 후 자동 복원하지 않으며 Windows 자체 WLAN 프로필은 삭제하지 않는다.

## Service Evidence and Vulnerability Claims

- 서비스 확인은 사전에 정한 포트, 호스트 수, 동시 실행 수, 시간 제한 안에서 수행하고 애플리케이션 payload를 전송하지 않는다.
- 열린 포트나 서비스 광고는 노출 근거이지 취약점 확정 근거가 아니다.
- 특정 CVE 판정은 정확한 제조사·모델·현재 버전과 공식 취약점 근거가 모두 있을 때만 허용한다.
- 공유기 관리 연동은 공식 API 또는 사용자가 승인한 검증된 인터페이스만 사용하며 인증 우회나 비밀번호 대입을 하지 않는다.

## Wi-Fi Scan Interface Boundary

- Windows의 현재 SSID 스캔은 활성 Native 802.11 인터페이스만 사용하고 Ethernet을 자동 대체 경로로 사용하지 않는다.
- Ethernet과 Wi-Fi가 같은 서브넷이어도 능동 probe의 출발 주소를 현재 Wi-Fi IPv4에 고정한다.
- 같은 게이트웨이와 서브넷을 공유하는 SSID는 L3 탐색 결과가 겹칠 수 있으므로 “SSID에 연결된 장비”로 단정하지 않고 “서브넷에서 관측된 장비”로 표시한다.
- SSID·밴드별 실제 접속 장비 판정은 사용자 인증을 거친 공유기 접속자 목록이나 공식 관리 API 근거가 있을 때만 제공한다.

## Graph Evidence Boundary

- 그래프 간선은 스캔 레코드에서 알고 있는 공유기-장비 관계만 표시한다.
- 서로 다른 게이트웨이나 관측 근거가 없는 노드 사이에 시각적 간선을 임의로 만들지 않는다.
- 장비 유형이 확인되지 않은 노드는 추측한 유형 색상 대신 `기타`로 표시한다.

## Device Identification Privacy

- MAC 주소나 IP 주소를 외부 제조사 조회 API에 보내지 않는다.
- 장비 이름과 유형은 로컬 링크에서 장비가 공개한 DNS-SD, SSDP, NetBIOS, 서비스 정보와 로컬 스캔 근거만 결합한다.
- 이름·서비스·공유기 관리 목록 등 제품 근거가 없으면 정확한 모델이나 유형을 단정하지 않는다.
- IEEE OUI 같은 제조사 레지스트리를 사용할 경우 전체 공개 데이터베이스를 로컬로 내려받아 조회하고 개별 MAC 주소를 외부로 전송하지 않는다.

## Dynamic Status Banner Consistency

- 화면 위에 동적으로 나타나는 진행, 성공, 중지, 경고, 오류 메시지는 동일한 공통 배너 컴포넌트를 사용한다.
- 모든 상태 배너는 배경을 반투명으로 표시하고 오른쪽에 닫기 버튼을 제공한다.
- 일반 배너는 반투명 검정과 흰색 글씨를 사용하고, 오류 배너는 반투명 오류색과 충분한 대비의 글씨를 사용한다.
- 배너 문구는 닫기 버튼과 독립적으로 전체 배너의 중앙에 정렬한다.
- 배너 상태 문구는 `검색 완료`, `신규 장비 1개 발견`, `연결 실패`처럼 짧은 명사형으로 끝낸다.
