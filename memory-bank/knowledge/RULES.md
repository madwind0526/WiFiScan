# Rules

## Project Independence

- WifiScan은 기존 프로젝트와 분리된 새 프로젝트다.
- 다른 프로젝트에서는 폴더 구조와 검증된 작업 절차만 참고한다.
- 기능, 데이터 모델, 패키지, 비즈니스 규칙은 WifiScan 요구사항에 근거해 새로 결정한다.

## Source and UI

- 코드 주석은 영어만 사용한다.
- 사용자에게 보이는 UI 텍스트는 한국어로 작성한다.

## Discovery Completeness

- 단일 단말 스캔으로 모든 장비 탐지를 보장하지 않는다.
- 공유기, 서브넷, 서비스 탐색 결과를 결합하고 관측 소스, 신뢰도, 누락 가능성을 표시한다.
- 장비가 탐지되지 않았다는 사실을 안전하다는 의미로 표현하지 않는다.

## Safe Remediation

- 비밀번호 대입, 취약점 악용, 서비스 중단, 무단 패치를 구현하지 않는다.
- 자동 대응은 공식 관리 API, 사용자 인증, 변경 미리보기, 명시적 확인, 실패 복구 경로가 모두 있을 때만 허용한다.
- 제품과 버전이 확인되지 않은 장비에는 특정 취약점 판정을 확정하지 않는다.

## Android Permissions

- Android 17/API 37 이상에서 직접 LAN 통신을 사용하면 `ACCESS_LOCAL_NETWORK` 런타임 권한 또는 시스템 중개 탐색 경로를 사용한다.
- Wi-Fi API 권한은 Android 버전과 실제 호출 API별로 분리하고 거부/철회 상태를 정상 흐름으로 처리한다.

## Active Discovery Limits

- 큰 서브넷은 무제한 탐색하지 않는다.
- Windows Wave 2 능동 탐색은 현재 주소가 속한 최대 `/24`와 254개 호스트로 제한하고 UI에 범위 제한을 표시한다.
- 식별자(IP, MAC 등)는 테스트와 로그에 평문으로 출력하지 않는다.

## Inventory Storage

- 장비 스냅샷은 기본적으로 로컬 애플리케이션 지원 디렉터리에만 저장한다.
- 첫 검색은 기준선이며 신규 장비 경고를 발생시키지 않는다.
- 로컬 스냅샷은 무제한으로 쌓지 않고 최근 30개만 유지한다.

## Remediation Safety

- 공식 공유기 커넥터가 없으면 자동 차단·격리·설정 변경을 실행하지 않고 수동 안내만 제공한다.
- 커넥터 실행에는 사용자 명시 승인, 변경 미리보기, 확인 문구, 복구 가능성이 필요하다.

## Android Verification

- Android 권한 요청 전 한국어 근거 화면을 제공한다.
- 권한 거부/철회 상태에서도 앱이 종료되지 않고 안내 상태를 유지한다.
- 실기기 미연결 시 Android APK 빌드와 플랫폼 채널 코드 검증만으로 완료를 주장하지 않는다.
